Highlight #1 Verison Control

Version control is, in a short description, basically just figuring out and controlling each seperate version of any sort of files that you may have changed or edited while working on a project. Version Control is just another way to keep track of everything. Because of version controlling, there are many different websites and companies that have made it possible for you to seamlessly scroll through and review anything you may have changed and wanted to redo. Websites like git make it extremely easy for you do this with a team on any number of projects or work related documents.

Highlight #2 Repositories

There are two types of repositoies in a verson control system. The first type of repositories is a remote repository. A remoter repository resides on a remote server that is shared among muliple team members. The second type is a local repository, which is a repository that resides on a local machine of an individual user. You can use a repository to control history. When a repository is created with files deveoplers can record their change history and start tracking their status and versions.

Highlight #3 Why use Git?

Git is very useful in the web application development feild. Git has a distrubted version control system. This allows every user to have a complete copy of the repository data stored locally on their machine. Git allows proccess to run much faster.Git also prevents you from accidentally going back and overwriting any files that you might need to refer back to. Git is a great tool to have when you are workiong on team projects and have many people many changes to a file.

Highlight #4 Git Commands

Git has many commands that are very useful. Some of the basic commands are git init, git add --all, git log, and git status. You also have some very useful remote commands as well which are git remote, git pull, and git push. Git add allows you to send out certain files that you have edited already and send them into a waiting bay. Git commit command pushes files that you have edited out onto the local repository. After that is done then you would use the git push command which pushes the files onto the remote repository for everyone to see.

Highlight #5 Version Control System Benefits

There are many benefits to a version control system. One of the benefits is that files cannont be overwritten. Verison control system is constantly creating new save points for the file to prevent this. Another benefit is that people are able to work on the same files simultaneously without conficts. With a verison control system you are able to revert back to an older verison of the file if needed. When using a VCS there is a common repository that holds all the latest fileds for the project.
